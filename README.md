# 勉強借金 — StudyDebt

> やるべき勉強を「借金」として可視化する勉強計画アプリ。
> 締切までにこなさないと、未消化分が **複利で利息** を膨らませて円換算で襲ってくる。
> 先延ばしの痛みを「数字」に変える、ちょっとひねくれた勉強アプリ。

## これは何

普通の学習計画アプリは「これからやること」を前向きに並べる。StudyDebt は逆で、
**やってない勉強＝負債** とみなし、放置するほど雪だるま式に金額が増える。

- 科目・タスクごとに「総量（ページ/問/時間）」と「締切」を登録
- 残量 × ¥100 を基準負債とし、本来のペースより遅れると **8%/日の複利** で利息が乗る
- 「+1 やった / +10」で返済（= 進捗）し、負債を減らす
- 完済すると緑の `完済 ✓`

## 動かし方

`index.html` を開くだけ。ビルド不要・依存ゼロ・完全クライアントサイド。
データはブラウザの `localStorage` のみに保存され、サーバーには一切送信されません。

GitHub Pages にそのまま置けば公開できます（Settings → Pages → ブランチを指定）。

## 設計メモ（利息モデル）

```
基準負債   = 残量 × ¥100
遅れ日数   = 締切超過日数 + (本来終わっているべき量との差 ÷ 1日あたりの想定量)
負債       = 基準負債 × (1 + 0.08) ^ 遅れ日数
今日の最低返済 = 残量 × ¥100 ÷ 締切までの残り日数
```

定数（`YEN_PER_UNIT`, `DAILY_INTEREST`）は `index.html` 冒頭で調整可能。

## 収益化について（正直版）

このコードは**動くプロトタイプ**であって、それ自体がお金を生むわけではありません。
$15,000/年 のような売上は、結局のところ「ユーザー獲得 × 課金導線 × 継続率」次第で、
そこは人間（あなた）の運用が必要です。現実的な筋道だけ置いておきます:

- **無料 → Pro 課金**: クラウド同期、複数端末、通知リマインド、テーマ、統計ダッシュボード
- **粗利**: 静的ホスティングなのでサーバー原価ほぼゼロ。Pro が月¥500なら **約250人**で年商150万円規模
- **配布**: App名で受験/資格界隈（SNS・受験コミュニティ）に「先延ばしが金額で殴ってくるアプリ」として刺す
- **検証が先**: まずこの無料版を出し、継続して使う人がいるか（＝痛みが効いているか）を見てから課金を足す

> 数字を保証するものではありません。アイデアと実装の出発点です。
